# English Easy Go - 國小英語單字王遊戲設計

## 概要

基於宜蘭縣 114 學年度國小英語單字王 400 字題庫，製作互動式單字測驗遊戲。目標部署平台為 Gemini Canvas。

## 技術架構

- **單一 HTML 檔案**：所有 CSS/JS 內嵌，相容 Gemini Canvas
- **Vanilla JS SPA**：用 DOM 操作實現頁面切換，不使用框架
- **400 字資料**：直接寫成 JS 物件陣列內嵌
- **AI 語意檢查**：Gemini Canvas 內建 Gemini model（開發階段用字串比對模擬）
- **排行榜**：Gemini Canvas 環境提供 Firestore（開發階段用 localStorage 模擬）
- **練習進度**：localStorage 儲存

## 頁面流程

```
首頁（輸入玩家名稱）
  ├── 練習模式
  │    ├── 單字選擇頁（全部 400 字列表，按字母分組，可勾選）
  │    ├── 選擇題型（中翻英 / 英翻中 / 選擇題）
  │    └── 練習遊戲頁
  └── 挑戰模式
       ├── 選擇題型（中翻英 / 英翻中 / 選擇題）
       ├── 選擇題數（10 / 20 / 30）
       ├── 挑戰遊戲頁
       └── 結算頁（分數 + 排行榜）
```

## 頁面設計

### 1. 首頁

- 遊戲標題「English Easy Go 單字王」
- 玩家名稱輸入框（記住上次輸入的名稱）
- 兩個大按鈕：練習模式、挑戰模式
- 可愛卡通風格，鮮豔色彩

### 2. 練習模式 - 單字選擇頁

- 上方搜尋框：即時篩選單字
- 字母快速跳轉按鈕列（A B C D ... Z）
- 長列表顯示全部 400 字，按字母分組
  - 每組標題顯示字母 + 已選/總數（如 `— A (3/8) —`）
  - 每組有全選/取消全選按鈕
  - 每個單字一行：勾選框 + 英文 + 中文
- 底部固定列：已選 N / 400 個單字 + 「開始練習」按鈕
- 記住上次的勾選狀態

### 3. 練習模式 - 題型選擇

- 三個題型卡片：中翻英、英翻中、選擇題
- 每個卡片顯示該題型剩餘需練習的單字數
- 已全部答對的題型顯示完成標記

### 4. 練習遊戲頁

- 頂部：進度條（已完成 / 總數）+ 需練習數
- 中間：題目區域
- 底部：輸入框或選項按鈕
- 答對：綠色勾勾 + 彈跳動畫，自動下一題
- 答錯：顯示正確答案，紅色提示，手動下一題
- 已答對的「單字+題型」組合不再出題
- 全部答對時顯示恭喜畫面

### 5. 挑戰模式 - 設定頁

- 題型選擇（中翻英 / 英翻中 / 選擇題）
- 題數選擇（10 / 20 / 30）
- 開始挑戰按鈕

### 6. 挑戰遊戲頁

- 頂部：題號（第 N / M 題）+ 目前得分
- 中間：題目區域
- 底部：輸入框或選項按鈕
- 答對/答錯動畫同練習模式
- 從 400 字中隨機抽題，不重複

### 7. 結算頁

- 本次成績：得分 / 總題數、正確率
- 排行榜（該題型 + 題數的 Top 10）
- 重新挑戰 / 回首頁按鈕

## 資料結構

### 單字資料

```js
const WORDS = [
  { id: 1, en: "afraid", zh: "害怕的" },
  { id: 2, en: "afternoon", zh: "下午" },
  // ...共 400 筆
];
```

### 練習進度（localStorage）

```js
// key: "vocabGame_practice_{playerName}"
{
  "afraid_zh2en": true,     // 中翻英已答對
  "afraid_en2zh": true,     // 英翻中已答對
  "afraid_choice": true,    // 選擇題已答對
  "afternoon_zh2en": true,
  // ...
}
```

### 單字選擇記錄（localStorage）

```js
// key: "vocabGame_selection_{playerName}"
[1, 2, 5, 10, ...]  // 選中的單字 id 陣列
```

### 排行榜（Firestore / localStorage）

```js
// collection: "leaderboard"
{
  playerName: "小明",
  score: 18,
  total: 20,
  quizType: "zh2en",   // "zh2en" | "en2zh" | "choice"
  timestamp: 1234567890
}
```

## 題型邏輯

### 中翻英

- 顯示：中文解釋
- 輸入：英文單字
- 判定：忽略大小寫、前後空格，完全比對
- 特殊處理：含空格的單字（如 "cell phone"）允許多餘空格

### 英翻中

- 顯示：英文單字
- 輸入：中文意思
- 判定：Gemini AI 語意檢查（開發階段用包含比對模擬）
- AI Prompt：「正確答案是「{answer}」，學生回答「{input}」，意思是否相同或足夠接近？只回答 YES 或 NO」

### 選擇題

- 顯示：隨機決定顯示英文或中文
- 選項：正確答案 + 3 個隨機干擾項，順序隨機
- 判定：選對即正確
- 干擾項選擇邏輯：從同字母開頭或相似類別的單字中選取

## 視覺風格

- **可愛卡通風**：鮮豔色彩（主色調藍/綠/橘）、圓角卡片、大字體
- **動畫效果**：答對彈跳、答錯搖晃、頁面切換淡入淡出
- **回饋機制**：答對綠色勾勾 + 鼓勵文字、答錯紅色叉叉 + 顯示正確答案
- **排行榜**：獎盃圖示（金銀銅）、表格排列

## 開發 vs 部署差異

| 功能 | 開發階段 | Gemini Canvas |
|------|---------|---------------|
| AI 語意檢查 | 字串包含比對 | Gemini model |
| 排行榜儲存 | localStorage | Firestore |
| 檔案結構 | 單一 index.html | 同左 |

開發時預留接口，搬到 Canvas 時只需替換兩個函式：
- `checkSemanticMatch(answer, input)` → 接 Gemini
- `saveScore(data)` / `getLeaderboard(filter)` → 接 Firestore
